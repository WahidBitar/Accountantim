# Story 3.11: OpenAPI-drift gate baseline

Status: ready-for-dev

## Story

As a **developer preventing accidental API-contract breakage**,
I want the **OpenAPI-drift gate baselined against Epic 2 + Epic 3 real endpoints**,
so that **future PRs changing the API surface fail CI until the `openapi.json` snapshot is regenerated and reviewed**. The generated spec from the running API is diffed against the committed snapshot via Schemathesis (or openapi-diff); any drift fails the build with a message pointing to the drift location; regenerating + committing the snapshot resolves the failure; an ESLint rule `no-handwritten-shared-contracts` forbids any TypeScript file under `contracts/` not marked as generated.

**Covers requirements:** ADR-005 (TS contracts generator + no handwritten shared contracts), architecture §3.3 (CI-drift gates), Epic 0 Story 0.8 deferral inventory (OpenAPI drift-gate baseline deferred to this story).

## Acceptance Criteria

### AC1 — Baseline openapi.json committed

**Given** Epic 2 + Epic 3 real endpoints have shipped
**When** the running API's source-gen OpenAPI document is captured
**Then** `tools/openapi/openapi.json` is committed to the repo as the baseline snapshot
**And** the snapshot reflects all POST/GET/PATCH/DELETE routes from:
  - `/v1/contacts` + sub-paths (Stories 2.1-2.5 + 3.4 + 3.8)
  - `/v1/journal-entries` + sub-paths (Stories 3.1-3.3 + 3.8)
  - `/v1/dashboard` + `/v1/dashboard/chart/*` (Stories 3.5-3.7)
  - `/v1/export/csv` (Story 3.9)
  - `/v1/users/me/*` (Story 3.1 T9 dependency)

### AC2 — Drift-detection tool in CI

**Given** the tool `tools/validate-openapi.ps1` (or `.sh` equivalent for Linux runners)
**When** CI runs the script
**Then** the script:
  1. Boots the API in ephemeral mode (using Testcontainers Postgres)
  2. Fetches the generated OpenAPI from `GET /openapi/v1.json`
  3. Diffs against the committed `tools/openapi/openapi.json` snapshot using **openapi-diff** (Atlassian OpenAPI-diff or the Node-based `openapi-diff` — decision: Atlassian for JVM parity in CI; if CI runner lacks JVM, fall back to the Node-based OSS tool)
  4. Exits 0 on no drift; exits non-zero on drift with a human-readable diff report
**And** the script runs as part of a required CI job `openapi-drift-gate`

### AC3 — Schemathesis contract property tests

**Given** Schemathesis is installed in the CI runner
**When** the job runs `schemathesis run tools/openapi/openapi.json --base-url=http://localhost:5000 --checks all`
**Then** Schemathesis exercises each endpoint with property-based test cases derived from the schema
**And** any response violating the schema (extra/missing fields, wrong types) fails the build
**And** the Schemathesis run completes in < 10 minutes at current endpoint count

### AC4 — Regeneration workflow

**Given** a developer intentionally changes an endpoint contract
**When** they run `tools/openapi/regenerate-snapshot.ps1`
**Then** the script boots the API, fetches the OpenAPI, and overwrites `tools/openapi/openapi.json`
**And** the developer commits the updated snapshot as part of their PR
**And** the PR description MUST include a brief "Contract change rationale" section (enforced via PR template — soft gate)

### AC5 — ESLint rule `no-handwritten-shared-contracts`

**Given** the frontend workspace `frontend/contracts/` directory (for generated client types)
**When** ESLint runs in CI
**Then** any file under `frontend/contracts/` without a leading comment `// Generated by NSwag — do not edit` (or equivalent marker) fails lint
**And** the rule is implemented as a custom ESLint rule in `tools/eslint-rules/no-handwritten-shared-contracts.js`
**And** registered in `eslint.config.js`
**And** app-local typed wrappers in `libs/core/api/*.client.ts` are exempt — the rule targets `contracts/` specifically

### AC6 — Documentation in CI failure message

**Given** the drift-gate fails
**When** the CI log is inspected
**Then** the log includes:
  - The specific endpoint(s) where drift was detected
  - Added / removed / modified fields
  - A pointer to the regeneration script: "Run `tools/openapi/regenerate-snapshot.ps1` and commit the updated snapshot if this is intentional"
  - Link to ADR-005 for context

### AC7 — Story 0.8 inventory update

**Given** Story 0.8 deferred the OpenAPI-drift baseline to this story
**When** this story completes
**Then** the Story 0.8 context file is annotated with a `## Status Update` noting the deferral resolved
**And** the Epic 0 deferred-items inventory reflects all four fitness gates + drift gate as shipped

## Dev Notes

### Critical guardrails

1. **`openapi.json` is the source of truth for the TS contract generator.** NSwag (per delegated decision) consumes this file to generate frontend clients. Breaking the file breaks the FE build.
2. **Drift-gate is required.** If this check becomes optional (branch-protection misconfig), contract breakage slips into main.
3. **Schemathesis runs property-based tests.** A passing OpenAPI diff + failing Schemathesis run means the runtime behavior disagrees with the schema (handler regression). Both must pass.
4. **Regenerate + commit is a deliberate act.** PR template's "Contract change rationale" prompt ensures the change is justified.
5. **ESLint rule targets `contracts/` only.** The `libs/core/api/*.client.ts` thin-wrapper escape hatch from ADR-005 stays valid; it's not a shared contract — it's app-local.
6. **Baseline MUST be committed before gate activation** — otherwise the first CI run trivially passes (empty diff).
7. **Schemathesis timeout budget: 10 minutes.** At current endpoint count this is generous; as endpoints grow, consider parallelization via `schemathesis run --workers=4`.

### CI workflow fragment (reference)

```yaml
openapi-drift-gate:
  runs-on: ubuntu-latest
  services:
    postgres:
      image: postgres:17
      env:
        POSTGRES_PASSWORD: postgres
      ports: [5432:5432]
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-dotnet@v4
      with:
        dotnet-version: '10.x'
    - uses: actions/setup-python@v5
      with:
        python-version: '3.12'
    - run: pip install schemathesis
    - run: pwsh tools/openapi/validate-openapi.ps1
    - run: schemathesis run tools/openapi/openapi.json --base-url=http://localhost:5000 --checks all
```
